# CodeSight for Java

> Java 代码深度洞察 VS Code 扩展 - 为 AI Agent 提供调用链提取与 LSP 代码分析能力

## 简介

**CodeSight for Java** 是一个 VS Code 扩展，通过复用已运行的 [Language Support for Java(TM) by Red Hat](https://marketplace.visualstudio.com/items?itemName=redhat.java) 扩展的 LSP 能力，提供两大类功能：

1. **调用链提取**：将方法调用链及相关代码切片导出为结构化上下文文件，供 AI Agent 直接读取
2. **LSP 代码分析**：跳转定义、查找引用、符号列表、查找实现、悬停信息——均通过命令面板/右键菜单触发

> 与 `jdt-lsp-cli` 的关系：功能对等，实现层不同。本扩展委托 VS Code 内置命令路由，无需独立启动 JDT LS 进程。

## 前置要求

- **VS Code** 1.85+（或 Qoder）
- **Red Hat Java 扩展**：[redhat.java](https://marketplace.visualstudio.com/items?itemName=redhat.java)（本扩展声明为依赖，会自动提示安装）
- **Java 项目**：已由 redhat.java 完成索引（工作区打开后等待左下角语言状态就绪）

## 安装

从 Open VSX Registry 搜索 `CodeSight for Java` 安装，或直接安装 `.vsix` 文件：

```bash
code --install-extension codesight-for-java-1.0.1.vsix
```

## 命令概览

所有命令均可通过以下方式触发：
- **右键菜单**（在 Java 文件编辑器中右键）
- **命令面板**（`Ctrl+Shift+P` → 输入 `Java AI:`）

| 命令 | 标题 | 说明 |
|------|------|------|
| `javaExtractor.extractCallChain` | Java AI: Extract Call Chain for AI | 提取完整调用链，导出上下文文件 |
| `javaExtractor.extractCoreCallChain` | Java AI: Extract Core Call Chain (Compact) | 精简模式：过滤日志/测试类，非入口方法只保留签名 |
| `javaExtractor.goToDefinition` | Java AI: Go to Definition | 跳转到符号定义位置 |
| `javaExtractor.findReferences` | Java AI: Find All References | 查找符号的所有引用，输出到 Output 面板 |
| `javaExtractor.showSymbols` | Java AI: Show File Symbols | 显示文件符号列表，支持点击跳转 |
| `javaExtractor.findImplementations` | Java AI: Find Implementations | 查找接口/抽象方法的所有实现 |
| `javaExtractor.showHover` | Java AI: Show Hover Info | 获取符号的类型声明与 Javadoc |

## 命令详解

### 1. extractCallChain — 提取调用链（完整模式）

分析当前光标所在方法，递归追踪其调用链，提取所有涉及方法的完整方法体及成员变量，生成上下文文件。

**使用方法：**
1. 在 Java 文件中，将光标放在目标方法名上
2. 右键 → `Java AI: Extract Call Chain for AI`
3. 显示进度通知，完成后提示打开文件

**输出文件：**
- 路径：`<workspace>/.skill_context/<ClassName>_<methodName>.java`（可在设置中修改）
- 格式：包含 import + 类定义 + 所有被调用方法体的 Java 代码切片

**输出示例：**
```java
// Generated by CodeSight for Java
// Entry: OrderService.processOrder
// Methods: 8 | Fields: 3

import com.example.domain.Order;
import com.example.repository.OrderRepository;

// === OrderService ===
public class OrderService {
  private final OrderRepository orderRepository;
  private final PaymentService paymentService;

  public Result processOrder(Order order) {
    validateOrder(order);
    Payment payment = paymentService.charge(order);
    return orderRepository.save(order, payment);
  }

  private void validateOrder(Order order) {
    if (order.getAmount() <= 0) throw new IllegalArgumentException("Invalid amount");
  }
}

// === PaymentService ===
public class PaymentService {
  public Payment charge(Order order) { ... }
}
```

---

### 2. extractCoreCallChain — 提取核心调用链（精简模式）

在完整模式基础上，应用更激进的过滤和压缩策略，使输出文件体积更小，适合 Token 预算有限的场景。

**与完整模式的区别：**

| 特性 | 完整模式 | 精简模式 |
|------|----------|----------|
| Javadoc | 移除（默认） | 移除 |
| 行内注释 | 移除（默认） | 移除 |
| 注解 | 保留（默认） | 全部移除 |
| 日志调用 | 保留 | **过滤**（log\*, debug\*, info\* 等） |
| 测试类 | 保留 | **过滤**（`**/test/**`，`*Test.java` 等） |
| 非入口方法 | 完整方法体 | **仅保留方法签名** |

**输出文件：**
- 路径：`<workspace>/.skill_context/<ClassName>_<methodName>_core.java`

---

### 3. goToDefinition — 跳转到定义

获取光标处符号（类、方法、变量、字段）的定义位置，并直接在编辑器中跳转。

**使用方法：**
1. 将光标放在符号名称上（如 `processOrder`、`OrderRepository`）
2. 右键 → `Java AI: Go to Definition`

**交互行为：**
- **单个结果**：直接打开目标文件并定位到定义行
- **多个结果**：弹出 QuickPick 列表，显示所有定义位置，选择后跳转

**QuickPick 示例：**
```
找到 2 个定义，选择跳转
  ↳ $(go-to-file) src/main/java/com/example/OrderService.java   Line 25
  ↳ $(go-to-file) src/main/java/com/example/OrderServiceImpl.java   Line 18
```

> 对应 LSP 协议：`textDocument/definition`  
> 对应 jdt-lsp-cli：`jls def <file> <line> <col>`

---

### 4. findReferences — 查找所有引用

查找光标处符号在整个项目中的所有引用位置，结果输出到 Output 面板（`Java Call Chain Extractor`）。

**使用方法：**
1. 将光标放在符号名称上
2. 右键 → `Java AI: Find All References`
3. 显示进度通知，完成后自动打开 Output 面板

**Output 面板输出示例：**
```
===== 引用查找结果 (共 5 处) =====
光标位置: /project/src/main/java/com/example/OrderService.java:25

[1] src/main/java/com/example/controller/OrderController.java:48:12
[2] src/main/java/com/example/controller/OrderController.java:62:8
[3] src/main/java/com/example/service/BatchService.java:101:20
[4] src/test/java/com/example/OrderServiceTest.java:33:5
[5] src/test/java/com/example/OrderServiceTest.java:67:5
```

> 注意：结果**包含声明本身**（`includeDeclaration: true`）  
> 对应 LSP 协议：`textDocument/references`  
> 对应 jdt-lsp-cli：`jls refs <file> <line> <col>`

---

### 5. showSymbols — 显示文件符号列表

获取当前 Java 文件中的所有符号（类、方法、字段、构造器、枚举等），以扁平化 QuickPick 列表展示，选择后跳转到对应行。

**使用方法：**
1. 打开任意 Java 文件（光标位置不限）
2. 右键 → `Java AI: Show File Symbols`

**QuickPick 展示格式：**
```
OrderService.java 中共 12 个符号，选择跳转
  $(symbol-class) OrderService                    Class   Line 15
  $(symbol-method) processOrder                   Method  Line 28  (in OrderService)
  $(symbol-field) orderRepository                 Field   Line 18  (in OrderService)
  $(symbol-method) validateOrder                  Method  Line 45  (in OrderService)
  $(symbol-constructor) OrderService              Constructor  Line 22  (in OrderService)
```

> 对应 LSP 协议：`textDocument/documentSymbol`  
> 对应 jdt-lsp-cli：`jls sym <file> --flat`

---

### 6. findImplementations — 查找实现

查找接口方法或抽象方法的所有具体实现类，在 QuickPick 中展示，选择后跳转。

**使用方法：**
1. 将光标放在**接口方法**或**抽象方法**名称上
2. 右键 → `Java AI: Find Implementations`

**交互行为：**
- **单个实现**：直接跳转到实现类
- **多个实现**：弹出 QuickPick 列表选择

**QuickPick 示例：**
```
找到 3 个实现，选择跳转
  $(symbol-class) src/main/java/com/example/JdbcOrderRepository.java   Line 22
  $(symbol-class) src/main/java/com/example/MongoOrderRepository.java  Line 15
  $(symbol-class) src/main/java/com/example/CacheOrderRepository.java  Line 30
```

> 如果光标不在接口/抽象方法上，会提示「未找到实现（可能不是接口或抽象方法）」  
> 对应 LSP 协议：`textDocument/implementation`  
> 对应 jdt-lsp-cli：`jls impl <file> <line> <col>`

---

### 7. showHover — 显示悬停信息

获取光标处符号的完整类型声明和 Javadoc 文档，输出到 Output 面板。

**使用方法：**
1. 将光标放在符号名称上
2. 右键 → `Java AI: Show Hover Info`

**Output 面板输出示例：**
```
===== Hover 信息 =====
位置: /project/src/main/java/com/example/OrderService.java:28:10

```java
public Result processOrder(Order order)
```

处理订单业务逻辑，包含校验、支付、持久化三个阶段。

@param order 订单对象，不可为 null
@return 处理结果，包含订单 ID 和支付流水号
@throws IllegalArgumentException 当订单金额非法时抛出
```

> 对应 LSP 协议：`textDocument/hover`  
> 对应 jdt-lsp-cli：`jls hover <file> <line> <col>`

## 配置项

通过 VS Code 设置（`Ctrl+,` → 搜索 `javaExtractor`）调整：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `javaExtractor.maxDepth` | `10` | 调用链最大递归深度 |
| `javaExtractor.outputDir` | `.skill_context` | 上下文文件输出目录（相对于工作区根目录） |
| `javaExtractor.removeJavadoc` | `true` | 移除 Javadoc 注释，减少 Token 数 |
| `javaExtractor.removeComments` | `true` | 移除方法体内的行内注释 |
| `javaExtractor.removeAnnotations` | `false` | 移除 `@Override`、`@Deprecated` 等注解（保留 `@Transactional` 等） |
| `javaExtractor.compactImports` | `true` | 仅保留提取方法实际用到的 import |
| `javaExtractor.shortImplPaths` | `true` | 实现类路径仅显示类名，不显示全限定名 |
| `javaExtractor.indentSize` | `2` | 缩进大小（0=不缩进，2 或 4） |
| `javaExtractor.compactMode` | `false` | 超精简模式：非入口方法只保留签名 |
| `javaExtractor.excludePatterns` | `["**/test/**", ...]` | 排除的文件路径模式 |
| `javaExtractor.excludeClassPatterns` | `["Log", "Logger", ...]` | 排除的类名模式（日志、断言类等） |
| `javaExtractor.excludeMethodPatterns` | `["toString", "log*", ...]` | 排除的方法名模式 |

## 与 jdt-lsp-cli 对比

| 维度 | jdt-lsp-cli | CodeSight for Java |
|------|-------------|-------------------|
| 运行环境 | 独立命令行，可脱离 IDE | VS Code 扩展，依赖 IDE |
| LSP 调用方式 | 直接启动 JDT LS 进程，使用 LSP 协议 | 委托 VS Code 内置命令路由给 redhat.java |
| 适用场景 | AI Agent 自动化、CI 脚本 | 开发者交互式操作、IDE 内调用 |
| 输出形式 | JSON（结构化，适合程序处理） | QuickPick / Output 面板（适合人机交互） |
| 调用链提取 | 仅基础调用层级 | **完整方法体 + 成员变量 + 上下文文件** |

## 常见问题

### Q: 命令执行后提示"未找到定义/引用"
确认 redhat.java 扩展已完成项目索引。观察 VS Code 左下角语言状态图标，等待显示就绪后再操作。

### Q: 调用链提取结果为空
- 确认光标放在**方法名**上（不是括号或空白）
- 尝试先触发一次 `showSymbols` 确认 LSP 已就绪

### Q: 输出目录 `.skill_context` 在哪里
位于工作区根目录下，可通过 `javaExtractor.outputDir` 配置修改。建议将该目录加入 `.gitignore`。

### Q: 精简模式和完整模式应该如何选择
- 向 AI 提问时 Token 充足 → 完整模式（保留更多上下文）
- Token 预算紧张或只关心核心逻辑 → 精简模式

## 技术架构

```
java-call-chain-extractor (CodeSight for Java)
├── src/
│   ├── extension.ts           # 扩展入口，注册全部 7 个命令
│   ├── lspFeatures.ts         # LSP 功能封装（定义/引用/符号/实现/悬停）
│   ├── callChainExtractor.ts  # 调用链递归提取核心逻辑
│   ├── fieldAnalyzer.ts       # 成员变量依赖分析
│   ├── outputGenerator.ts     # 上下文文件生成器
│   ├── codeOptimizer.ts       # Token 优化（注释/注解/import 裁剪）
│   └── types.ts               # 类型定义
└── out/                       # 编译输出（TypeScript → JavaScript）
```

### LSP 命令映射

| VS Code 内置命令 | LSP 协议方法 | jdt-lsp-cli 等价命令 |
|-----------------|-------------|---------------------|
| `vscode.executeDefinitionProvider` | `textDocument/definition` | `jls def` |
| `vscode.executeReferenceProvider` | `textDocument/references` | `jls refs` |
| `vscode.executeDocumentSymbolProvider` | `textDocument/documentSymbol` | `jls sym` |
| `vscode.executeImplementationProvider` | `textDocument/implementation` | `jls impl` |
| `vscode.executeHoverProvider` | `textDocument/hover` | `jls hover` |
